# Weapon Visuals Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the plain colored-sphere projectiles for all 5 SWIV3D weapons with distinct hand-authored shapes (rocket, missile, plasma bolt, napalm canister), add muzzle flash for the special weapons, and give each weapon its own impact effect instead of one generic flash.

**Architecture:** Prefabs for 4 weapon bodies live under `Assets/Resources/Prefabs/Weapons/` and are built live in the connected Unity Editor via ProBuilder MCP tools. `Projectile.CreateRuntime()` loads a prefab by weapon-type name via `Resources.Load`, falling back to the existing primitive-sphere path if none is found — so the game is playable after every single task. A new `WeaponVisuals.cs` owns all particle-driven muzzle-flash and impact effects, dispatching on `WeaponType`, reusing `ExplosionUtil`'s existing particle-burst and building-blast helpers rather than duplicating them.

**Tech Stack:** Unity 6 (Built-in Render Pipeline — no URP/HDRP package installed), C#, ProBuilder (already in `Packages/manifest.json`), Unity Editor MCP tools (`mcp__ai-game-developer__*`) via the live connected instance (`Swiv3D-remastered`).

## Global Constraints

- No new fields on `WeaponDefinition` — visual choices key off `WeaponType` directly (per `docs/superpowers/specs/2026-07-22-weapon-visuals-design.md`).
- No gameplay stat changes — damage, ammo, homing behavior, explosion radii, burn stats are untouched. This is visuals-only.
- Prefabs live at exactly `Assets/Resources/Prefabs/Weapons/<WeaponType>.prefab` (e.g. `Rockets.prefab`, matching `WeaponType.Rockets.ToString()`), so `Resources.Load<GameObject>("Prefabs/Weapons/" + type)` resolves them.
- No textures — flat/emissive `Standard`-shader materials only, consistent with the project's existing no-texture-art style.
- Missing prefab must fall back to the current primitive-sphere path — never a hard failure.
- No automated test suite covers gameplay scripts in this project (confirmed: no `Assets/**/Tests*` folder exists). Verification is manual playtesting through the connected Unity Editor MCP session: enter Play Mode, fire the weapon, capture a screenshot, check `console-get-logs` for errors.
- Every 3D shape/orientation step below gives a concrete starting value. Where exact ProBuilder shape-orientation math can't be verified ahead of running the tool, the step includes an explicit visual-verification sub-step with a bounded correction (try the alternate sign) — this is normal practice for live 3D authoring, not a placeholder.

```csharp
    /// <summary>Loads the weapon's authored prefab from Resources, or builds a colored sphere as a fallback.</summary>
    public static Projectile CreateRuntime(WeaponDefinition def, Vector3 position, Vector3 direction, Transform owner) {
        var go = BuildVisualRoot(def.type);
        go.name = def.type + "Projectile";
        go.transform.position = position;
        RemoveAllColliders(go);
        var projectile = go.AddComponent<Projectile>();
        projectile.Initialize(def, owner, direction);
        return projectile;
    }

    static GameObject BuildVisualRoot(WeaponType type) {
        var prefab = Resources.Load<GameObject>("Prefabs/Weapons/" + type);
        if (prefab != null) {
            return Object.Instantiate(prefab);
        }
        return GameObject.CreatePrimitive(PrimitiveType.Sphere);
    }

    static void RemoveAllColliders(GameObject root) {
        var colliders = root.GetComponentsInChildren<Collider>();
        for (int i = 0; i < colliders.Length; i++) {
            Object.Destroy(colliders[i]);
        }
    }
```

The explicit `AddComponent<SphereCollider>()` is no longer needed — `Projectile` already carries `[RequireComponent(typeof(SphereCollider))]` (line 7), which auto-adds it the moment `AddComponent<Projectile>()` runs; `Initialize()` (lines 27-29) already configures it (`isTrigger = true; radius = 0.5f;`).

Read `Assets/Scripts/WeaponVisuals.cs`, replace the two empty method bodies with:
```csharp
using UnityEngine;

/// <summary>
/// Procedural muzzle-flash and impact effects for each weapon type.
/// Projectile bodies are authored prefabs (see Projectile.CreateRuntime); this class only
/// handles the particle/light bursts around firing and hitting, reusing ExplosionUtil where possible.
/// </summary>
internal static class WeaponVisuals {
    internal static void SpawnMuzzleFlash(WeaponType type, Transform firePoint) {
        if (type == WeaponType.Plasma || type == WeaponType.SmartBomb) {
            return;
        }
        Vector3 pos = firePoint.position;
        ExplosionUtil.SpawnParticleBurst(
            "MuzzleSmoke", pos,
            burstCount: 10, startSpeed: 3f, startSize: 0.5f, lifetime: 0.3f, gravity: -0.1f, sphereRadius: 0.15f,
            startColor: new Color(0.9f, 0.9f, 0.85f, 0.8f),
            midColor: new Color(0.6f, 0.6f, 0.6f, 0.4f),
            endColor: new Color(0.4f, 0.4f, 0.4f, 0f),
            sizeMulEnd: 1.8f, destroyAfter: 0.5f
        );
        SpawnLightPop(pos, new Color(1f, 0.85f, 0.5f), 6f, 6f, 0.15f);
    }

    internal static void SpawnImpact(WeaponType type, Vector3 point, WeaponDefinition def, Transform owner) {
        switch (type) {
            case WeaponType.Rockets:
            case WeaponType.HomingMissiles:
                SpawnFireballImpact(point, def);
                break;
            default:
                break;
        }
    }

    static void SpawnFireballImpact(Vector3 point, WeaponDefinition def) {
        ExplosionUtil.SpawnFlash(point, def.explosionRadius, def.projectileColor);
        ExplosionUtil.SpawnParticleBurst(
            "RocketSmoke", point,
            burstCount: 14, startSpeed: 4f, startSize: 1.2f, lifetime: 0.8f, gravity: -0.2f, sphereRadius: 0.4f,
            startColor: new Color(0.3f, 0.28f, 0.25f, 0.7f),
            midColor: new Color(0.2f, 0.18f, 0.16f, 0.4f),
            endColor: new Color(0.1f, 0.1f, 0.1f, 0f),
            sizeMulEnd: 2f, destroyAfter: 1.6f
        );
        ExplosionUtil.SpawnParticleBurst(
            "RocketSparks", point,
            burstCount: 10, startSpeed: 10f, startSize: 0.15f, lifetime: 0.4f, gravity: 1f, sphereRadius: 0.25f,
            startColor: new Color(1f, 0.8f, 0.3f, 1f),
            midColor: new Color(1f, 0.4f, 0.05f, 0.9f),
            endColor: new Color(0.3f, 0.05f, 0f, 0f),
            sizeMulEnd: 0.2f, destroyAfter: 0.9f
        );
    }

    static void SpawnLightPop(Vector3 pos, Color color, float intensity, float range, float lifetime) {
        var go = new GameObject("MuzzleFlashLight");
        go.transform.position = pos;
        var light = go.AddComponent<Light>();
        light.type = LightType.Point;
        light.color = color;
        light.intensity = intensity;
        light.range = range;
        light.shadows = LightShadows.None;
        var fade = go.AddComponent<ExplosionLightFade>();
        fade.lifetime = lifetime;
        fade.startIntensity = intensity;
    }
}
```
